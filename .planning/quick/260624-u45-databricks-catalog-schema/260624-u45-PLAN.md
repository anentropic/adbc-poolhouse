<objective>
Fix `DatabricksConfig.to_adbc_kwargs()` silently dropping `catalog` and `schema_`
in decomposed mode (req DBX-02). The decomposed branch builds
`databricks://token:{encoded}@{host}:443{http_path}` and returns `{"uri": uri}`,
never reading `self.catalog` (_databricks_config.py:64) or `self.schema_` (:67).
Unqualified table/view refs then fail with `TABLE_OR_VIEW_NOT_FOUND`. The fix
appends URL-encoded `catalog`/`schema` to the DSN query string — the Databricks
Go SQL driver parses `?catalog=&schema=` from the DSN even though the ADBC-level
option is unsupported.

Purpose: Make Databricks default namespace honor the config, matching Snowflake
(`adbc.snowflake.sql.schema`) and Trino (`catalog`/`schema` kwargs).
Output: Fixed config method, unit tests, updated docstring + guide.
</objective>
