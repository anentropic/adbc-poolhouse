# Phase 21.1 Plan 01: Dispatch fix and unit test Summary

Three-shape ADBC dispatch lands in `create_adbc_connection`, closing the
`TypeError: missing 1 required positional argument: 'uri'` that broke
`create_pool()` for Quack, PostgreSQL, and FlightSQL across the last three
releases. A dispatch-level unit test pins the new Family A' branch so the
regression class cannot return.

## What changed

`src/adbc_poolhouse/_driver_api.py` — the inner `db_kwargs` branch now splits
into Family A (Snowflake / BigQuery, uri optional or absent) and Family A'
(PostgreSQL / FlightSQL / Quack, uri required-positional). The new branch
fires when all three conditions hold on `inspect.signature(mod.connect)`:

1. `"db_kwargs"` is in the parameters
2. `"uri"` is in the parameters
3. `params["uri"].default is inspect.Parameter.empty`

When the branch fires, `kwargs.pop("uri")` extracts the URI and the call
becomes `mod.connect(uri_val, db_kwargs=kwargs)`. `db_kwargs` is passed by
name (never positionally) because Quack's `connect()` declares it
KEYWORD_ONLY — passing positionally would break Quack at runtime while
appearing to work in dispatch.

Family B (DuckDB / SQLite) and the Foundry-driver path are untouched. The
function signature is unchanged; no new top-level imports were added; the
existing `# type: ignore` suppressions stay attached to their original lines.

`tests/test_driver_api.py` — appended
`test_family_a_prime_pops_uri_when_required_positional` to
`TestDbApiModuleSignatureDispatch`. The stub uses an inline
`def connect(uri: str, *, db_kwargs=None)` literal so
`inspect.signature(connect)` reports the affected shape. This is the
deliberate counter-pattern to bare `MagicMock` patches, which were exactly
the gap that hid this bug for three releases.

## Decisions made

1. **Three-branch dispatch in a single function, not a registry or
   strategy class.** The plan and RESEARCH.md were explicit on this — any
   per-driver mapping would re-introduce the registration pattern that
   v1.2.0 deliberately removed.
2. **`db_kwargs=` always passed by name.** Quack's KEYWORD_ONLY shape
   would reject a positional call; PostgreSQL and FlightSQL accept either,
   so by-name is the safest superset.
3. **`kwargs.pop("uri")` raises on missing key.** No guard. A config that
   triggers the Family A' branch but does not include `"uri"` is a real
   config-shape bug and should surface immediately rather than silently
   mis-dispatch (RESEARCH.md Assumption A2 and Anti-Patterns).
4. **Test stub is a real `def connect(...)` literal**, not `MagicMock(spec=...)`.
   The literal is unambiguously inspectable and does not depend on the
   upstream driver being installed at test-collection time. Mirrors the
   pre-existing pattern at `tests/test_driver_api.py:19,36`.
