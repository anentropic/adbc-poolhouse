# Roadmap: v1.2.0

**Milestone:** Plugin/Extensibility API  
**Target:** Enable 3rd party libraries to add backend providers via entry points  
**Previous:** v1.1.0 — 12 backends, pytest-adbc-replay integration tests  

---

## Goals

1. **Manual Registration**: `register_backend()` API for in-project backends
2. **Auto-Discovery**: Entry point hooks for 3rd party libs to register automatically
3. **Documentation**: Guide for plugin authors on implementing custom backends

---

## Quality Gates

All phases must pass before completion:
- Unit tests for all new public APIs
- Integration tests using dummy backend plugin
- `uv run basedpyright` passes (strict mode)
- `uv run ruff check` passes
- `uv run ruff format --check` passes

### Phase 1: Driver Import Semi-Integration Tests

- TEST-01, TEST-02, TEST-03, TEST-04

2. Mock assertions verify correct ADBC connection kwargs
3. Justfile recipe for installing all drivers locally

**Status**: Completed 2026-03-12

---

### Phase 2: Registry Infrastructure

**Goal**: Add a backend registry and manual registration API

**Depends on**: Phase 1

**Requirements**: 
- REG-01, REG-02, REG-03, TEST-INFRA-01

**Success Criteria** (what must be TRUE):
  1. `register_backend(name, config_class, translator, driver_package, ...)` registers a new backend at runtime
  2. Registered backends work identically to built-in backends (pool creation, driver resolution, translation)
  3. Dummy backend fixture created for integration testing
  4. Attempting to register a duplicate name raises `BackendAlreadyRegisteredError`

**Status**: Completed 2026-03-12

---

### Phase 2.5: Translator Consolidation

**Goal**: Consolidate translator functions into config class methods using Pydantic v2 features

**Depends on**: Phase 2

**Requirements**:
- Refactor for plugin interface consistency

**Success Criteria** (what must be TRUE):
  1. All 12 built-in config classes have `to_adbc_kwargs()` method
  2. Pydantic field aliases handle ADBC key prefixes
  3. Custom serializers handle bool→string and SecretStr→string transformations
  4. Databricks URI construction via `@model_validator`
  5. Registry stores driver_path only, translator is built-in to config class
  6. All existing tests pass unchanged

### Phase 3: Entry Point Discovery

**Goal**: Auto-discover 3rd party backends via Python entry points

**Depends on**: Phase 2.5

**Requirements**:
- EP-01, EP-02, EP-03, TEST-INFRA-02, TEST-INFRA-03

**Success Criteria** (what must be TRUE):
  1. 3rd party packages can declare backends via `[project.entry-points."adbc_poolhouse.backends"]`
  2. Entry points are discovered lazily on first `create_pool()` call (not at import time)
  3. Entry point load failures produce clear error messages identifying the offending package
  4. CI installs dummy backend and runs integration tests
  5. Malformed plugin fixtures verify graceful error handling
