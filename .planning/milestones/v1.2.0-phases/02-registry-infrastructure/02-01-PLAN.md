<task type="auto" tdd="true">
  <name>Task 2: Create registry module with registration API</name>
  <files>src/adbc_poolhouse/_registry.py, tests/test_registry.py</files>
  <behavior>
    - Test 1: register_backend() with valid params succeeds (no exception)
    - Test 2: register_backend() with duplicate name raises BackendAlreadyRegisteredError
    - Test 3: register_backend() with None config_class raises TypeError
    - Test 4: register_backend() with non-callable translator raises TypeError
    - Test 5: get_translator() returns correct translator for registered backend
    - Test 6: get_driver_path() returns correct driver_path for registered backend
    - Test 7: get_translator() for unregistered config raises BackendNotRegisteredError
  </behavior>
  <action>
    Create `src/adbc_poolhouse/_registry.py` implementing the registry pattern from RESEARCH.md:
    
    1. Type aliases:
       - `TranslatorFunc = Callable[[WarehouseConfig], dict[str, str]]`
       - `_Registration = tuple[type[WarehouseConfig], TranslatorFunc, str]`
    
    2. Module-level storage (private):
       - `_registry: dict[str, _Registration] = {}` — forward lookup by name
       - `_config_to_name: dict[type[WarehouseConfig], str] = {}` — reverse lookup by config class
       - `_lazy_registrations: dict[type[WarehouseConfig], Callable[[], None]] = {}` — lazy registration functions
    
    3. Public `register_backend(name: str, config_class: type[WarehouseConfig], translator: TranslatorFunc, driver_path: str) -> None`:
       - Validate name not already in `_registry` → raise `BackendAlreadyRegisteredError(name)`
       - Validate `config_class` is a type → raise `TypeError(f"config_class must be a class, got {type(config_class).__name__}")`
       - Validate `translator` is callable → raise `TypeError(f"translator must be callable, got {type(translator).__name__}")`
       - Store in `_registry[name]` and `_config_to_name[config_class]`
    
    4. Public `get_translator(config: WarehouseConfig) -> TranslatorFunc`:
       - Look up config type in `_config_to_name`, then get translator from `_registry`
       - Raise `BackendNotRegisteredError(config_type.__name__)` if not found
    
    5. Public `get_driver_path(config: WarehouseConfig) -> str`:
       - Same lookup pattern, return driver_path
       - Raise `BackendNotRegisteredError` if not found
    
    6. Public `ensure_registered(config: WarehouseConfig) -> None`:
       - If config type in `_config_to_name`, return (already registered)
       - If config type in `_lazy_registrations`, call the lazy function
       - Otherwise raise `BackendNotRegisteredError`
    
    7. Internal `register_lazy(config_class: type[WarehouseConfig], registration_func: Callable[[], None]) -> None`:
       - Store in `_lazy_registrations[config_class]`
    
    Use `TYPE_CHECKING` block for `WarehouseConfig` import to avoid circular imports.
    
    Add `TestRegisterBackend` and `TestRegistryLookup` test classes to `tests/test_registry.py`.
  </action>
  <verify>
    <automated>uv run pytest tests/test_registry.py -x</automated>
  </verify>
  <done>Registry module complete with register_backend, get_translator, get_driver_path, ensure_registered; all tests pass; basedpyright passes</done>
</task>
