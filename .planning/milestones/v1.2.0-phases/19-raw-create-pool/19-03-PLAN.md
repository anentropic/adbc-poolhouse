<objective>
Fix dbapi_module connect() signature incompatibility between PyPI ADBC drivers.

Purpose: `_driver_api.py` line 80 calls `mod.connect(db_kwargs=kwargs)` assuming all ADBC dbapi modules accept `db_kwargs`. DuckDB and SQLite have `connect(path_or_uri, **kwargs)` signatures that do NOT accept `db_kwargs`. This causes `TypeError: Connection.__init__() got an unexpected keyword argument 'db_kwargs'` when using `dbapi_module` with DuckDB or SQLite.

Output: `create_adbc_connection()` introspects the target module's `connect()` signature to dispatch correctly for both families.
</objective>

With signature-aware dispatch:

```python
if dbapi_module is not None:
    import inspect

    mod = importlib.import_module(dbapi_module)
    sig = inspect.signature(mod.connect)
    if "db_kwargs" in sig.parameters:
        conn = mod.connect(db_kwargs=kwargs)  # type: ignore[no-any-return]
    else:
        conn = mod.connect(**kwargs)  # type: ignore[no-any-return]
    return conn  # type: ignore[return-value]
```

Move `import inspect` to the top of the file (module-level import) rather than inline. This keeps the import clean per project conventions.
