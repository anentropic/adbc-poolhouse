Completely rewrite `tests/test_drivers.py`. The old file tests `resolve_driver()` from `_drivers.py` which no longer exists. The new file tests `config._driver_path()` and `config._dbapi_module()` directly.

New test structure:

```
Module docstring: "Unit tests for config _driver_path() and _dbapi_module() methods."
```

**class TestDuckDBDriverPath:**
- `test_duckdb_calls_resolve_with_driver_path_method_name`: Mock `importlib.util.find_spec` to return a spec, mock `__import__` to return a package with `driver_path()` (not `_driver_path()`). Call `DuckDBConfig()._driver_path()`. Assert it returns the mocked path. This proves DuckDB uses `method_name="driver_path"`.
- `test_duckdb_fallback_when_package_missing`: Mock `importlib.util.find_spec` to return None. Call `DuckDBConfig()._driver_path()`. Assert returns `"adbc_driver_duckdb"` (manifest fallback string).
- `test_duckdb_dbapi_module_returns_none`: `assert DuckDBConfig()._dbapi_module() is None`

**class TestPyPIDriverPath:**
- `test_snowflake_found_returns_driver_path`: Mock `find_spec` non-None + mock `__import__` with `_driver_path()`. Assert returns mocked path.
- `test_snowflake_missing_returns_package_name`: Mock `find_spec` None. Assert returns `"adbc_driver_snowflake"`.
- `test_bigquery_missing_returns_package_name`: Mock `find_spec` None. Assert returns `"adbc_driver_bigquery"`.
- `test_postgresql_missing_returns_package_name`: Mock `find_spec` None. Assert returns `"adbc_driver_postgresql"`.
- `test_flightsql_missing_returns_package_name`: Mock `find_spec` None. Assert returns `"adbc_driver_flightsql"`.
- `test_sqlite_missing_returns_package_name`: Mock `find_spec` None. Assert returns `"adbc_driver_sqlite"`.

**class TestPyPIDbApiModule:**
- `test_snowflake_installed_returns_dbapi_module`: Mock `find_spec` non-None on `"adbc_driver_snowflake"`. Assert `SnowflakeConfig(account="a")._dbapi_module() == "adbc_driver_snowflake.dbapi"`.
- `test_snowflake_not_installed_returns_none`: Mock `find_spec` to return None. Assert `._dbapi_module() is None`.
- `test_bigquery_installed_returns_dbapi_module`: Similar.
- `test_postgresql_installed_returns_dbapi_module`: Similar with `"adbc_driver_postgresql.dbapi"`.
- `test_flightsql_installed_returns_dbapi_module`: Similar with `"adbc_driver_flightsql.dbapi"`.
- `test_sqlite_dbapi_module_always_none`: No mock needed. `assert SQLiteConfig()._dbapi_module() is None` -- with comment explaining incompatible signature.

**class TestFoundryDriverPath:**
- `test_databricks_returns_short_name`: `DatabricksConfig(uri=SecretStr(...))._driver_path() == "databricks"`. No find_spec mock needed (method doesn't call it).
- `test_redshift_returns_short_name`: `RedshiftConfig()._driver_path() == "redshift"`
- `test_mysql_returns_short_name`: `MySQLConfig(host="h", user="u", database="db")._driver_path() == "mysql"`
- `test_clickhouse_returns_short_name`: `ClickHouseConfig(host="h", username="u")._driver_path() == "clickhouse"`
- `test_trino_returns_short_name`: `TrinoConfig()._driver_path() == "trino"`
- `test_mssql_returns_short_name`: `MSSQLConfig()._driver_path() == "mssql"`
- `test_foundry_dbapi_module_all_none`: Test all 6 foundry configs return None for _dbapi_module(). Use `@pytest.mark.parametrize` if desired.

**class TestCustomConfigContract (3P-CONTRACT):**
- `test_custom_protocol_config_has_required_methods`: Create a minimal class (NOT subclassing BaseWarehouseConfig) that implements to_adbc_kwargs() and _driver_path(). Verify it satisfies `isinstance(config, WarehouseConfig)` using the Protocol check. This tests the 3rd-party contract.

**Keep class TestCreateAdbcConnectionFoundryNotFound** unchanged from the existing file -- these tests are for `_driver_api.py` which is not changing. Copy them verbatim.

**Mocking strategy change**: Mock `importlib.util.find_spec` on the config's OWN module, not on the global. For `_driver_path()`, the mock target depends on where the call happens:
- For configs calling `self._resolve_driver_path()`, the `find_spec` call is in `_base_config.py`, so mock `importlib.util.find_spec` (global target).
- For configs calling `importlib.util.find_spec` directly in `_dbapi_module()`, mock `importlib.util.find_spec` (same global target).

No `clean_registry` fixture needed anywhere -- there is no registry state to clean.
