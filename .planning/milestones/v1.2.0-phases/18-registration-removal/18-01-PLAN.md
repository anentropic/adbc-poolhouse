- `SnowflakeConfig._driver_path()`: `return self._resolve_driver_path("adbc_driver_snowflake")`
- `SnowflakeConfig._dbapi_module()`: check `importlib.util.find_spec("adbc_driver_snowflake")` -- if not None, return `"adbc_driver_snowflake.dbapi"`, else `None`. Add `import importlib.util` at top of file.

- `PostgreSQLConfig`: same pattern with `"adbc_driver_postgresql"`
- `FlightSQLConfig`: same pattern with `"adbc_driver_flightsql"`

**PyPI special cases:**
- `SQLiteConfig._driver_path()`: `return self._resolve_driver_path("adbc_driver_sqlite")`
- `SQLiteConfig._dbapi_module()`: return `None` explicitly with comment `# SQLite dbapi has incompatible signature (positional uri, no db_kwargs)`
- `DuckDBConfig._driver_path()`: `return self._resolve_driver_path("adbc_driver_duckdb", method_name="driver_path")` -- note the different method_name because adbc_driver_duckdb uses `driver_path()` not `_driver_path()`

**Foundry drivers (static strings, no find_spec):**
- `DatabricksConfig._driver_path()`: `return "databricks"`
- `DatabricksConfig._dbapi_module()`: return `None`
- `RedshiftConfig._driver_path()`: `return "redshift"` / `_dbapi_module()`: `None`

- `MySQLConfig._driver_path()`: `return "mysql"` / `_dbapi_module()`: `None`
