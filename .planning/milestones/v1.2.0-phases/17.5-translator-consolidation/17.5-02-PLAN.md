  <behavior>
    - DuckDBConfig.to_adbc_kwargs() returns {path: database, access_mode: "READ_ONLY" if read_only}
    - Boolean read_only field → str.lower() for "true"/"false"
    - Test assertions unchanged from translate_duckdb tests
  </behavior>
  <action>
    Implement `to_adbc_kwargs()` in DuckDBConfig:
    
    DuckDB is the simplest backend — it only has two fields:
    - `database: str` (maps to `"path"` key, not `"database"`)
    - `read_only: bool` (maps to `"access_mode": "READ_ONLY"` when True)
    
    Key insight from translator: The field names don't match ADBC keys directly.
    - `database` → `"path"` (DJ: DuckDB uses 'path' not 'database')
    - `read_only` → `"access_mode": "READ_ONLY"` only when True
    
    Implementation:
    ```python
    def to_adbc_kwargs(self) -> dict[str, str]:
        """Convert DuckDBConfig to ADBC driver kwargs.
        
        Returns:
            Dict with 'path' key (always) and 'access_mode' when read_only=True.
        """
        result: dict[str, str] = {"path": self.database}
        if self.read_only:
            result["access_mode"] = "READ_ONLY"
        return result
    ```
    
    Note: DuckDB is too simple for Field aliases. The mapping is too different
    (database→path), so manual construction is cleaner.
    
    Update tests in test_translators.py:
    - TestDuckDBTranslator tests now call config.to_adbc_kwargs()
    - Keep assertions identical

  <behavior>
    - ClickHouseConfig.to_adbc_kwargs() handles both URI mode and decomposed mode
    - URI mode: return {uri: secret_value}
    - Decomposed mode: return {username, host, port, password, database} with None omitted
    - port is always str(port) even though it's an int field
    - Test assertions unchanged from translate_clickhouse tests
  </behavior>
  <action>
    Implement `to_adbc_kwargs()` in ClickHouseConfig:
    
    Read _clickhouse_translator.py first to understand both modes:
    
    Mode 1 - URI mode (uri is set):
    - Return {"uri": uri.get_secret_value()}
    
    Mode 2 - Decomposed mode:
    - Return individual kwargs: username, host, port (as string), password, database
    - Omit None values
    
    Key insight: port is an int but must be output as string (str(self.port)).
    
    Implementation:
    ```python
    def to_adbc_kwargs(self) -> dict[str, str]:
        """Convert ClickHouseConfig to ADBC driver kwargs.
        
        Supports two modes:
        - URI mode: {uri: ...} when uri is set
        - Decomposed mode: {username, host, port, ...} otherwise
        """
        if self.uri is not None:
            return {"uri": self.uri.get_secret_value()}
        
        # Decomposed mode
        result: dict[str, str] = {
            "username": self.username,
            "host": self.host,
            "port": str(self.port),
        }
        if self.password is not None:
            result["password"] = self.password.get_secret_value()
        if self.database is not None:
            result["database"] = self.database
        return result
    ```
    
    Add @field_serializer for SecretStr fields if not already present.

    - Keep assertions identical
