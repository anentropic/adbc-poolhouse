  <name>Task 2: Implement SnowflakeConfig.to_adbc_kwargs</name>
  <files>src/adbc_poolhouse/_snowflake_config.py, tests/test_translators.py</files>
  <behavior>
    - Config.to_adbc_kwargs() returns same dict as translate_snowflake(config)
    - Uses Field(alias="adbc.snowflake.sql.*") for ADBC key prefixes
    - @field_serializer for boolean fields → 'true'/'false'
    - @field_serializer for SecretStr fields → get_secret_value()
    - @field_serializer for Path fields → str()
    - schema_ field → `"adbc.snowflake.sql.schema"` key (validation_alias="schema", alias="adbc.snowflake.sql.schema")
  </behavior>
  <action>
    Update SnowflakeConfig to implement `to_adbc_kwargs()` using Pydantic v2 features:
    
    1. Add Field aliases for ADBC-prefixed keys:
       - `account: str` → add `Field(alias="adbc.snowflake.sql.account")` 
         (but wait — Snowflake translator shows account has no prefix in current code, only in kwargs)
       - Actually, looking at translator: `account` → `"adbc.snowflake.sql.account"`
       - So use: `account: str = Field(alias="adbc.snowflake.sql.account")`
         But wait — the alias is for serialization OUTPUT, not input validation.
         Need to be careful: Pydantic's `alias` is for serialization output, `validation_alias` is for input.
         For this refactor: we want field names in Python, ADBC keys in output.
         So: `alias="adbc.snowflake.sql.account"` means `model_dump(by_alias=True)` outputs that key.
    
    2. Handle special cases that don't fit Field alias pattern:
       - `user` field → `"username"` key (different name)
       - Use override in to_adbc_kwargs for non-aliasable mappings
    
    3. Add @field_serializer decorators:
       - For boolean fields (tls_skip_verify, ocsp_fail_open_mode, keep_session_alive, disable_telemetry, cache_mfa_token, store_temp_creds):
         ```python
         @field_serializer(
             "tls_skip_verify", "ocsp_fail_open_mode", "keep_session_alive",
             "disable_telemetry", "cache_mfa_token", "store_temp_creds",
             mode="plain"
         )
         def serialize_bool(self, value: bool) -> str:
             return str(value).lower()
         ```
       - For SecretStr fields (password, private_key_pem, private_key_passphrase, oauth_token, okta_url is NOT SecretStr):
         Actually, okta_url is str, not SecretStr. SecretStr fields need:
         ```python
         @field_serializer(
             "password", "private_key_pem", "private_key_passphrase", "oauth_token",
             mode="plain"
         )
         def serialize_secret(self, value: SecretStr | None) -> str | None:
             if value is None:
                 return None
             return value.get_secret_value()
         ```
       - For Path fields (private_key_path):
         ```python
         @field_serializer("private_key_path", mode="plain")
         def serialize_path(self, value: Path | None) -> str | None:
             if value is None:
                 return None
             return str(value)
         ```
    
    4. Keep model_dump(exclude_none=True, by_alias=True) as base, then merge in special cases:
       - Fields that don't use alias pattern (like user→username)
       - Fields that need custom logic
    
    CRITICAL: The current SnowflakeConfig has fields with `validation_alias="schema"` but no alias for output.
    Need to also set `alias="adbc.snowflake.sql.schema"` for output serialization.
    
    Implementation approach:
    - Use model_dump(mode="json") to trigger SecretStr serializers
    - Merge manually for fields that need different key names
    - Exclude pool-related fields (pool_size, max_overflow, timeout, recycle)
    
    Update test_translators.py tests to use config.to_adbc_kwargs() directly:
    - Add new test class TestSnowflakeToAdbcKwargs
    - Tests call config.to_adbc_kwargs() instead of translate_snowflake(config)
    - Keep assertions the same (output dict should be identical)
