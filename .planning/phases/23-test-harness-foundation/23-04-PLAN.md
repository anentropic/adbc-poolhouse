CRITICAL FILE LOCATION (blocker fix): this self-test file lives at `tests/_async_harness/test_harness.py` — INSIDE the conftest's package — NOT at `tests/test_async_harness.py`. pytest conftest fixtures propagate DOWNWARD only (own dir + subdirs), never to a parent or sibling. The `anyio_backend` fixture is defined in `tests/_async_harness/conftest.py` (Plan 01); a self-test file that is a sibling of that directory cannot see it and every `@pytest.mark.anyio` test would fail collection with `fixture 'anyio_backend' not found`. Placing the file inside the package fixes this. A `test_*.py` file inside the package is still collected by pytest regardless of the `__init__.py` marker, and it now sits at/below the conftest so it sees `anyio_backend`. Do NOT hoist `anyio_backend` into the root `tests/conftest.py` to "fix" this — that re-introduces RESEARCH Pitfall 4.
