<objective>
Lock in the already-shipped PEP 562 import guard (PKG-02/03, D-01) with a subprocess-isolated
regression test. Phase 24/25 already implemented the `__getattr__` lazy import + the
`ImportError` naming `[async]` in `src/adbc_poolhouse/__init__.py` ‚Äî this plan adds tests that
prove it, it does NOT rewrite the guard. The test spawns a child Python process that installs a
`MetaPathFinder` raising `ImportError` for any `anyio` import (simulating anyio-absent without
uninstalling it), then asserts (a) `import adbc_poolhouse` succeeds and `create_pool` is present,
and (b) accessing `create_async_pool` raises an `ImportError` containing `[async]`.

Purpose: a permanent regression so a future refactor of `__getattr__` or `_LAZY_ASYNC_NAMES`
cannot silently break the zero-cost sync import or the helpful error message.
Output: new `tests/test_pkg_import_guard.py`.
</objective>
