<objective>
Tighten the async offload typing surface (PKG-05, D-03) so argument types are actually
checked at the dispatch boundary. Replace the loose `Callable[..., _T]` + `*args: object`
on both `offload()` (`_offload.py`) and `cancellable_offload()` (`_cancel.py`) with the
PEP 646 `TypeVarTuple` / `Unpack` variadic-forwarder pattern — `fn: Callable[[Unpack[_Ts]], _T]`
+ `*args: Unpack[_Ts]` — mirroring anyio's own `to_thread.run_sync` signature.

NOTE — mechanism correction (RESEARCH-verified): the requirement text and ROADMAP say
`ParamSpec`/`Concatenate`, but ParamSpec does NOT compile here — the typing spec forbids the
keyword-only `limiter`/`on_dispatch`/`on_abort` params from following `*args: P.args`
(basedpyright `reportGeneralTypeIssues`). `TypeVarTuple` is the correct, idiomatic mechanism
(it is what anyio uses). Using TypeVarTuple per RESEARCH (ParamSpec rejected — does not compile).

Purpose: real argument-type checking at the offload boundary while keeping basedpyright
strict at 0 errors.
Output: signature-only edits to `_offload.py` + `_cancel.py`; an expect-error typing fixture.
</objective>
