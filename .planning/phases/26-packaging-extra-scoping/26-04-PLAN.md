<objective>
Prove the zero-cost-sync claim with a dedicated CI job (PKG-04, D-04). Add a `sync-no-anyio`
job to `.github/workflows/ci.yml` that installs ONLY runtime deps + a sync backend (duckdb) +
pytest — excluding the `dev` dependency group, which is the only place anyio/trio/aiotools
live — asserts anyio is genuinely absent, then runs the sync suite with `tests/async`
deselected. This proves the shipped package has no hard async dependency.

Depends on Plan 01: the job runs `uv sync --locked`, which fails unless the `[async]` extra +
refreshed `uv.lock` from Plan 01 are already on the branch.

Purpose: a permanent CI guard that the sync path works with anyio uninstalled.
Output: edited `.github/workflows/ci.yml` (additive — a sibling job).
</objective>
