## Key Decisions

1. **Section placement and depth.** New heading is `#### Dispatch contract`,
   placed directly after the existing `_dbapi_module()` code block and
   immediately before `## Pool tuning`. The plan explicitly specifies
   four-hash depth so the subsection nests inside `### \`_dbapi_module()\``.

2. **Three rows in the order Family B → Family A → Family A'.** Matches the
   ordering in RESEARCH.md Example 5 so the simpler shapes appear first and
   the most constrained shape (required-positional `uri`) appears last, where
   the closing prose recommends against it for new drivers.

3. **No cross-refs in the new subsection.** The plan warns that strict-mode
   mkdocs fails on broken refs, and none of the surrounding text needs to
   point at API anchors here. The subsection is self-contained — only plain
   backtick inline code for identifiers.

4. **Sandbox workaround for the mkdocs gate.** `uv run mkdocs build --strict`
   was blocked by the sandbox's deny-write on `~/.cache/uv/sdists-v9/.git`.
   The workaround was to invoke `mkdocs` directly via
   `/Users/paul/Documents/Dev/Personal/adbc-poolhouse/.venv/bin/mkdocs` — the
   same interpreter that `uv run` would have used. The strict build still
   exercises the same configuration, plugin set, and gate.
