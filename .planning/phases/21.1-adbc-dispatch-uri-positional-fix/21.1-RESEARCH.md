# Phase 21.1: ADBC dispatch URI-positional fix - Research

**Researched:** 2026-05-20
**Domain:** ADBC PyPI-driver dispatch logic (`_driver_api.create_adbc_connection`)
**Confidence:** HIGH

## Summary

Phase 21.1 fixes a `TypeError: connect() missing 1 required positional argument: 'uri'` that breaks `create_pool()` for every PyPI driver whose `connect()` declares `uri` as a required positional AND has `db_kwargs` somewhere in its signature. The bug affects three backends: Quack (introduced in Phase 21), PostgreSQL (latent since v1.0.0), and FlightSQL (latent since v1.0.0). Snowflake and BigQuery are unaffected because their `uri` parameter has a default (`uri = None`), and BigQuery has no `uri` parameter at all.

The fix is a one-line dispatch branch in `_driver_api.py`. The test gap that hid this bug for three releases is that the semi-integration tests in `tests/test_driver_imports.py` patch `mod.connect` with a bare `MagicMock()`, which accepts any args/kwargs combination. `inspect.signature(MagicMock())` returns a wildcard signature (`(*args, **kwargs)`), so the existing dispatch code routes through the `db_kwargs in sig.parameters` branch and calls `MagicMock(db_kwargs=kwargs)` — which succeeds against a MagicMock but would fail against the real driver.

**Primary recommendation:** Three-task plan: (1) fix the dispatch logic and add the unit test against a faked module; (2) replace the bare `MagicMock` patches in the three affected `Test*Imports` classes with signature-preserving stubs and delete the duplicate `test_quack_returns_short_name`; (3) add the `_dbapi_module()` contract paragraph to `custom-backends.md`, build docs strict, run humanizer.

The bug is reproducible. Three lines below reproduced live against the installed drivers in this repo:

```

adbc_driver_postgresql.dbapi:  BUG REPRODUCED -- TypeError: connect() missing 1 required positional argument: 'uri'
adbc_driver_flightsql.dbapi:   BUG REPRODUCED -- TypeError: connect() missing 1 required positional argument: 'uri'
```

### Detection Rule (DISP-01)

The new branch fires when **all three** conditions hold for `inspect.signature(mod.connect).parameters`:

1. `"db_kwargs"` is a key in `sig.parameters`
2. `"uri"` is a key in `sig.parameters`
3. `sig.parameters["uri"].default is inspect.Parameter.empty` (required, no default)

When the branch fires:

```python
uri_val = kwargs.pop("uri")  # KeyError surfaces a config bug — fail loud
conn = mod.connect(uri_val, db_kwargs=kwargs)
```

The `kwargs.pop("uri")` (rather than `kwargs["uri"]`) is important: the original `kwargs` dict must not contain `"uri"` when passed as `db_kwargs=...`, because some ADBC drivers reject unknown keys inside `db_kwargs`. The dispatch handles the routing; the kwargs dict it passes onward is the original minus `"uri"`.

### Anti-Patterns to Avoid

- **Do not check `kwargs.get("uri")` first to decide the branch.** The signature is the source of truth, not the data. A future config that forgets to include `"uri"` would silently fall through to the old (broken) branch instead of failing loudly.
- **Do not introduce a global registry of "drivers that need positional uri".** The whole point of the signature-introspection dispatch is to avoid per-driver registration.
- **Do not change the `WarehouseConfig` Protocol.** The fix is downstream of the protocol; no config-side change is needed.
- **Do not silently swallow `KeyError` from `kwargs.pop("uri")`.** If the signature says `uri` is required but the config did not supply it, that is a real bug — let it surface.

## Common Pitfalls

### Pitfall 1: `MagicMock()` masks signature mismatches

**What goes wrong:** A test that does `with patch("foo.connect", return_value=mock_conn)` replaces `foo.connect` with a `MagicMock`, which accepts any args/kwargs. `inspect.signature(MagicMock())` returns a wildcard signature, so the dispatch code's branch selection is decided by the wildcard, not by the real driver's shape. The test passes; real-world execution fails.

**Why it happens:** `unittest.mock.MagicMock` is designed to be call-compatible with anything — that's its whole point. The trade-off is that signature-shape regressions slip past.

**How to avoid:** Use a real `def connect(...)` function literal whose parameter list matches the real driver's exactly. Patch with `new=` (or `side_effect=`) pointing to that real function. The existing `tests/test_driver_api.py::TestDbApiModuleSignatureDispatch` already does this — see lines 19 and 36.

**Warning signs:** A change to the dispatch logic in `_driver_api.py` does not break any `Test*Imports` test, even when a casual reading suggests it should. That is the symptom: the bare-`MagicMock` patches absorb everything.

### Pitfall 2: `kwargs` mutation across calls

**What goes wrong:** `kwargs.pop("uri")` mutates the dict in place. If `_pool_factory.py` ever stored or reused the kwargs dict across calls (e.g. for retry), the second call would see a different shape.

**Why it happens:** Python dicts are mutable by reference. The factory passes `resolved_kwargs` straight to `create_adbc_connection`, which currently treats it as owned.

**How to avoid:** Verify (a) `_create_pool_impl` calls `create_adbc_connection` exactly once per pool creation; (b) the dict is not reused elsewhere; (c) if at any future point reuse becomes a possibility, switch to `kwargs.pop("uri")` on a `kwargs.copy()` first. Today, **single-use is the current contract** — the simpler `pop` is appropriate.

**Warning signs:** Future retry logic added to `_pool_factory.py` silently drops `"uri"` on the second attempt.

### Pitfall 3: Confusing `KEYWORD_ONLY` vs `POSITIONAL_OR_KEYWORD`

**What goes wrong:** `adbc_driver_quack.dbapi.connect` declares `db_kwargs` as KEYWORD_ONLY (after a `*`); `adbc_driver_postgresql.dbapi.connect` declares it as POSITIONAL_OR_KEYWORD. The new fix passes `db_kwargs=kwargs` by name regardless, so both work. But if anyone ever changes the fix to positional (`mod.connect(uri_val, kwargs)`), Quack would break because `db_kwargs` is keyword-only.

**Why it happens:** Different driver authors made different signature choices. We do not control upstream.

**How to avoid:** Always pass `db_kwargs=kwargs` by name. Document this in the inline comment on the new branch.

**Warning signs:** Quack works in tests but fails at runtime with `TypeError: connect() takes 2 positional arguments but 3 were given`.

### Pitfall 4: Tests that patch the wrong module

**What goes wrong:** A test does `with patch("adbc_driver_quack.dbapi.connect", new=stub)` — but `stub` is a `MagicMock` again. The signature-preserving stub upgrade requires actually defining a real function with the exact shape.

**Why it happens:** Drive-by edit picks the wrong `new=` value.

**How to avoid:** The signature-preserving stub is a real `def connect(uri, *, db_kwargs=None, ...) -> MagicMock` — not a `MagicMock(spec=adbc_driver_quack.dbapi.connect)`. The `spec` variant works but is less explicit than a literal function and depends on the upstream driver being installed at test-collection time, which the conditional-mock pattern in `tests/test_driver_imports.py` is specifically designed to avoid.

**Warning signs:** Tests pass when the driver is installed but fail when it is not (or vice versa).

## Code Examples

### Example 1: The dispatch fix (DISP-01)

```python
# src/adbc_poolhouse/_driver_api.py — inside create_adbc_connection
# REPLACE the existing two-branch block:
#
#     if dbapi_module is not None:
#         mod = importlib.import_module(dbapi_module)
#         sig = inspect.signature(mod.connect)
#         if "db_kwargs" in sig.parameters:
#             conn = mod.connect(db_kwargs=kwargs)
#         else:
#             conn = mod.connect(**kwargs)
#         return conn
#
# WITH the three-branch block:

if dbapi_module is not None:
    mod = importlib.import_module(dbapi_module)
    sig = inspect.signature(mod.connect)
    params = sig.parameters
    if "db_kwargs" in params:
        uri_param = params.get("uri")
        if uri_param is not None and uri_param.default is inspect.Parameter.empty:
            # Family A' (Quack / PostgreSQL / FlightSQL): uri is required-positional.
            # Pop "uri" from kwargs and pass positionally; the remaining kwargs
            # ride as db_kwargs= so the driver does not reject them.
            uri_val = kwargs.pop("uri")
            conn = mod.connect(uri_val, db_kwargs=kwargs)
        else:
            # Family A (Snowflake / BigQuery): uri optional or absent.
            conn = mod.connect(db_kwargs=kwargs)
    else:
        # Family B (DuckDB / SQLite): no db_kwargs parameter.
        conn = mod.connect(**kwargs)
    return conn
```

### Example 2: Dispatch-level unit test (DISP-07)

```python
# tests/test_driver_api.py — append to TestDbApiModuleSignatureDispatch
def test_family_a_prime_pops_uri_when_required_positional(self) -> None:
    """Family A' (Quack/Postgres/FlightSQL): required-positional uri + db_kwargs."""
    call_record: dict[str, Any] = {}

    # Real function with the affected signature shape:
    # required-positional `uri` (no default), keyword `db_kwargs`.
    def connect(uri: str, *, db_kwargs: dict[str, str] | None = None) -> MagicMock:
        call_record["uri_positional"] = uri
        call_record["db_kwargs"] = db_kwargs
        return MagicMock()

    mock_mod = MagicMock()
    mock_mod.connect = connect

    with patch("importlib.import_module", return_value=mock_mod):
        create_adbc_connection(
            "",
            {"uri": "quack://h:1", "adbc.quack.token": "t"},
            dbapi_module="mock_family_a_prime",
        )

    # uri was popped and passed positionally; the rest stayed in db_kwargs
    assert call_record["uri_positional"] == "quack://h:1"
    assert call_record["db_kwargs"] == {"adbc.quack.token": "t"}
    assert "uri" not in (call_record["db_kwargs"] or {})
```

```python
# tests/test_driver_imports.py — replacement for the bare-MagicMock patch in
# TestQuackImports, TestPostgreSQLImports, TestFlightSQLImports (driver-installed branch)

class TestQuackImports:
    def test_create_pool_wiring(self) -> None:
        config = QuackConfig(uri="quack://h:1234")
        mock_conn = MagicMock()
        mock_conn.adbc_clone = MagicMock(return_value=MagicMock())

        # Signature-preserving stub: shape matches adbc_driver_quack.dbapi.connect
        # so inspect.signature() in _driver_api dispatch sees a required-positional uri.
        captured: dict[str, Any] = {}

        def stub_connect(uri: str, *, db_kwargs: dict[str, str] | None = None,
                         conn_kwargs: dict[str, str] | None = None,
                         autocommit: bool = False) -> MagicMock:
            captured["uri"] = uri
            captured["db_kwargs"] = db_kwargs
            return mock_conn

        if _driver_installed("adbc_driver_quack"):
            with patch("adbc_driver_quack.dbapi.connect", new=stub_connect):
                pool = create_pool(config)
                pool.dispose()
            assert captured["uri"] == "quack://h:1234"  # received positionally
            assert captured["db_kwargs"] == {}            # uri popped, no other kwargs
        else:
            with patch("adbc_driver_manager.dbapi.connect", return_value=mock_conn) as m:
                pool = create_pool(config)
                pool.dispose()
            m.assert_called_once()
            assert "driver" in m.call_args.kwargs

#### Dispatch contract

When `_dbapi_module()` returns a string, `create_pool()` imports that module
and inspects `connect()`'s signature to choose the call shape:

| Signature shape                                                                | Call form                              | Examples                              |
|--------------------------------------------------------------------------------|----------------------------------------|---------------------------------------|
| No `db_kwargs` parameter                                                       | `connect(**to_adbc_kwargs())`          | DuckDB, SQLite                        |
| Has `db_kwargs`, `uri` absent or with a default                                 | `connect(db_kwargs=to_adbc_kwargs())`  | Snowflake, BigQuery                    |
| Has `db_kwargs`, `uri` required-positional (no default)                         | `connect(uri, db_kwargs={...})` — `uri` popped from kwargs first | PostgreSQL, FlightSQL, Quack          |

If you ship an ADBC driver and want adbc-poolhouse to dispatch through it,
make sure your `connect()` signature falls into one of these three shapes.
The simplest safe choice is to keep `uri` optional (give it a default of
`None`) so the dispatcher does not need to pop it out of `db_kwargs`.

Return `None` from `_dbapi_module()` when the driver should route through
`adbc_driver_manager.dbapi.connect()` (the Foundry path). This is the right
default for native shared-library drivers without a Python `dbapi` submodule.
```
