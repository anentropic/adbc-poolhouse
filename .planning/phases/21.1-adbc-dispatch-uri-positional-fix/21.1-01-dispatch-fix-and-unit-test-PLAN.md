<objective>
Fix the dispatch bug in `_driver_api.create_adbc_connection` that breaks
`create_pool()` for any PyPI driver whose `connect()` declares `uri` as a
required positional parameter alongside a `db_kwargs` parameter (Quack,
PostgreSQL, FlightSQL). Add a dispatch-level unit test against a faked
module so the regression class is locked in.

Purpose: closes the live `TypeError: connect() missing 1 required positional
argument: 'uri'` that breaks the documented quickstart for three backends.
The fix is one new branch; the test pins the branch behaviour.

Output: edited `_driver_api.py` with a three-branch dispatch block, plus a
new test method on `TestDbApiModuleSignatureDispatch`.
</objective>
